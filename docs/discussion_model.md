# Discussion Model

## Purpose

This document defines the conceptual model by which Agentic Co-Emergence
transforms a fixed collection of independent perspectives into an
evolving collective understanding.

The objective of the runtime is **not** to simulate conversation. It is
to model how understanding changes through governed interaction.

------------------------------------------------------------------------

# 1. Fundamental Principle

The primary computational object is **understanding**, not dialogue.

Dialogue is the observable mechanism through which understanding
evolves.

    Initial Understanding
            │
            ▼
     Governed Discussion
            │
            ▼
    Revised Understanding

The transcript is evidence of this transformation, not the
transformation itself.

------------------------------------------------------------------------

# 2. Initial Conditions

Discussion begins with an immutable Perspective Pack.

Each perspective represents an agent's understanding prior to any
interaction.

No agent begins with knowledge of how its own perspective will change.

------------------------------------------------------------------------

# 3. Observable vs Latent State

Every contribution has two aspects.

## Observable

The contribution itself.

Examples:

-   argument
-   challenge
-   clarification
-   question
-   concession
-   synthesis

## Latent

The internal changes produced by the contribution.

Examples:

-   increased confidence
-   reduced confidence
-   new uncertainty
-   stronger agreement
-   refined concepts
-   revised priorities
-   new open questions

The runtime should model the latent changes.

------------------------------------------------------------------------

# 4. Understanding Delta

Every contribution produces an Understanding Delta (ΔU).

Conceptually:

U(t+1) = U(t) + ΔU

ΔU may include:

-   new claims
-   modified claims
-   strengthened evidence
-   weakened evidence
-   conceptual refinement
-   clarification of disagreement
-   creation of new questions

Not every contribution produces every kind of change.

------------------------------------------------------------------------

# 5. Levels of Understanding

Discussion operates simultaneously at three levels.

## Individual

Each agent maintains a mutable understanding.

## Relational

Relationships between agents evolve:

-   agreement
-   disagreement
-   trust
-   influence
-   acknowledged expertise

## Collective

The discussion acquires properties not attributable to any single
participant.

These include:

-   shared concepts
-   common vocabulary
-   unresolved tensions
-   emergent themes
-   candidate syntheses

Emergence occurs primarily at this level.

------------------------------------------------------------------------

# 6. Emergence

Emergence is not merely the appearance of a novel statement.

Emergence occurs when the collective understanding contains structure
that no individual perspective contained explicitly.

Examples include:

-   identifying hidden dimensions of the problem
-   revealing common assumptions
-   discovering a better framing question
-   integrating apparently conflicting perspectives

------------------------------------------------------------------------

# 7. Role of the Runtime

The runtime governs:

-   permissible operations
-   visibility of information
-   ordering of contributions
-   state transitions
-   update rules

The runtime does **not** determine the intellectual content of
contributions.

------------------------------------------------------------------------

# 8. Completion

A discussion ends because the runtime reaches a defined completion
condition, not because unanimity has been achieved.

Possible completion conditions include:

-   maximum rounds
-   stability of understanding
-   diminishing conceptual change
-   explicit termination criterion

The final synthesis is a snapshot of collective understanding at that
point in time.

------------------------------------------------------------------------

# Closing Principle

Agentic Co-Emergence is best understood as a computational model of
evolving understanding. Discussion is the mechanism; understanding is
the object.
